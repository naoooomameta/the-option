# theoption（jp.theoption.com）改修提案書

> 対象: 既存サイト https://jp.theoption.com/
> 作成: 2026-06-28
> 前提: 公式サイト本体は本環境のネットワークポリシーで直接閲覧不可（403）。本提案は **網羅調査（`docs/theoption-inventory.md`）＋ バイナリーオプション領域のCRO定石** に基づく。実装着手前にステージング環境での現状UI確認を推奨。

---

## 1. エグゼクティブサマリー

theoption の強み（**最大ペイアウト400% / 取引5方式 / 登録不要100万円デモ / BTC出金上限1億円 / FIUライセンス**）は、現状その多くが**第三者アフィリエイトサイト経由でしか伝わっていない**。公式サイトでこれらを正しく言語化し、**「登録不要デモ」を入口にしたノーリスク獲得ファネル**へ再設計することで、CVR改善余地が大きい。

優先順位は **Now（トップ再設計・5方式可視化・信頼設計）→ Next（IA統合・入出金最新化）→ Later（モバイルUX・計測/AB基盤）**。

## 2. 現状の主要課題（調査ベース）

| # | 課題 | 根拠 / 詳細 |
|---|---|---|
| C1 | **情報が4ドメインに分散** | 本体 `jp.theoption.com/ja-jp` ／取引 `www.theoption.com` ／ヘルプ `educate.theoption.com` ／ブログ `theoption-blog.com`。回遊分断・SEO評価の希薄化。 |
| C2 | **強みがトップで言語化不足** | 400%・5方式・100万円デモ・1億円出金などが、主に外部まとめ記事で初めて伝わる。 |
| C3 | **CV導線の摩擦** | ノーリスクの「登録不要デモ」が主CTA化されていない可能性。口座開設一本足になりがち。 |
| C4 | **信頼・リスクの先回り説明が薄い** | 分別管理あり/信託保全なし・FSA無登録など、不安要素への誠実な説明が不足＝離脱要因。 |
| C5 | **情報鮮度** | クレカ入金停止など仕様変更が周知されにくい（古い案内が残ると不信に直結）。 |
| C6 | **モバイル体験の作り込み** | 専用アプリ廃止→ブラウザ版推奨。一方「スマホはインジケーター非表示」等の制約がありUX設計が重要。 |

## 3. 改修方針（IA / UX / CRO / コンテンツ / 技術）

### 3.1 情報設計（IA）
- **教育/ブログを本体配下に集約**（例 `/learn`）。初心者教育 → デモ → 口座開設の動線を一本化し、SEOを統合。
- グローバルナビを「**取引方法 / プラットフォーム / 入出金 / キャンペーン / 学ぶ / 会社・安全性**」に整理。
- パンくず・関連リンクで回遊を設計。

### 3.2 トップページ（最重要）
- ヒーローで価値を要約：`最大400% / 最短15秒 / 5方式 / ¥0手数料`。
- **主CTA＝「登録不要でデモ」**（ノーリスク獲得）、副CTA＝「無料で口座開設」。
- 直下に**信頼バー**（FIU番号・分別管理・SSL・日本語サポート）。
- **5方式の比較表**で差別化を一目で（→ 本PRで試作済み）。
- プラットフォーム機能（4分割・トレーダーズチョイス・標準3インジ）を実機能ベースで訴求。

### 3.3 CRO（コンバージョン最適化）
- ファネル設計：**デモ開始 → 口座開設 → 本人確認 → 入金 → 初回取引**。各段の離脱を計測し改善。
- 追従CTA（モバイル下部固定）、Exit-Intent、社会的証明（実績数値・受賞歴があれば）。
- 口座開設フォームの項目最小化・進捗表示・離脱復帰メール。
- 本人確認の不安解消（必要書類・所要時間・セルフィー撮影のコツを事前提示）。

### 3.4 信頼・コンプライアンス設計
- 「**安全性**」ページを新設：分別管理の説明、SSL、KYC、そして**信託保全なし・FSA無登録の事実を誠実に開示**しつつ、リスク管理の考え方を提示。隠さないことが逆に信頼を生む。
- リスク警告を全ページ共通フッターで明示（既存方針を維持）。

### 3.5 コンテンツ／SEO
- 「バイナリーオプションとは」「ペイアウト率とは」「始め方」等の初心者導線を公式 `/learn` に内製化（現状は外部に流出）。
- 入出金・取引方式ページを**最新仕様で更新**（クレカ停止、bitwallet/銀行/仮想通貨、各上限）。
- 構造化データ（FAQPage, BreadcrumbList, Organization）を付与。

### 3.6 技術・計測
- パフォーマンス：画像最適化・遅延読み込み・Core Web Vitals。
- 計測：GA4イベント（demo_start / signup / kyc_complete / deposit / first_trade）＋ヒートマップ。
- **A/Bテスト基盤**でヒーローCTA・比較表・フォームを継続検証。

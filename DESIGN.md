# theOPTION — デザインシステム（案A「TERMINAL」）

> プロ仕様トレーディング端末の世界観。**漆黒 × ライム × モノスペース数値**。
> 精密・本格・高級。バイナリーオプションLPのコンバージョンと信頼訴求を担う。

このドキュメントは `styles.css` の **単一の正解（source of truth）** です。
新しい値を足すときは、まずここにトークンを定義してから CSS の `var(--…)` で参照します。

---

## 1. デザイン原則

1. **数値は主役** — ペイアウト率・価格・カウントダウンは等幅（IBM Plex Mono）で揃え、桁を信頼の根拠にする。
2. **静かな漆黒、一点のライム** — 背景は限りなく黒。アクセント `#CDFF4A` は CTA と「効いている数字」だけに使い、乱用しない。
3. **余白でグレードを出す** — 詰め込まず、8px グリッドのリズムで呼吸させる。
4. **動きは控えめに、意味を持って** — モーションは状態変化と登場の補助。`prefers-reduced-motion` を常に尊重。
5. **アクセシブルであること** — フォーカスリング・コントラスト・キーボード操作を最初から担保する。

## 6. モーション

| トークン | 値 |
|---|---|
| `--ease` | `cubic-bezier(.22,.61,.36,1)`（標準・減速） |
| `--ease-out` | `cubic-bezier(.16,1,.3,1)`（登場） |
| `--dur-fast` | 150ms |
| `--dur` | 240ms |
| `--dur-slow` | 600ms |

**既存アニメーション**
- `toMarquee` — ティッカー横流し（32s linear infinite）
- `toPulse` — ロゴ/ライブドットの鼓動（2.4s / 1.8s）
- `toBlink` — キャンペーンのライブ点滅（1.4s）
- `toReveal` — スクロール登場（下から `10px`・フェードイン）

**ルール**
- ホバー: ボタンは `translateY(-1px)` + 影、カードは境界を `--line-3` へ + `--shadow-soft`。
- 登場: `.reveal` をビューポート進入で `IntersectionObserver` が発火（JSなし環境でも内容は表示）。
- `@media (prefers-reduced-motion: reduce)` で全アニメーションと smooth-scroll を停止。

## 7. コンポーネント

### ボタン `.btn`
- `--accent`（塗り・900・`--accent-ink` 文字、ホバーで持ち上げ＋グロー）
- `--ghost`（境界線・透明、ホバーで境界明るく＋微背景）
- サイズ: `--lg`（16px / 16×28）, `--xl`（17px / 18×36）。
- フォーカス時 `outline: 2px solid --accent` 相当のリングを表示。

### HIGH / LOW ピル `.hl`
- HIGH = 緑グラデ＋緑境界、LOW = 赤グラデ＋赤境界。ラベルは太字。
- `--flat` 派生はワンクリック注文用（塗りなし・中央寄せ）。

### カード `.card` / `.step` / `.faq`
- `--panel` 面、`1px` 境界、`--r-lg`。ホバーで境界＋影を強調。

### チップ／タグ／ピル
- `.chip`（インジケーター名・極小ラベル）、`.tag`（BASIC/ADVANCED）、`.pill`（ライセンス・CAMPAIGN）。

### セクション枠
- `.section`（最大幅＋縦余白）、`.section--dark`（全幅 `--bg-deep`）、`.section--flush-top`（上余白を詰める）。
- 見出し: `.section__eyebrow`（mono・ライム・`NN / LABEL`）＋ `.section__title`。
